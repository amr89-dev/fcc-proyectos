Introducción a CSS

CSS acrónimo de Cascade Style Sheet, que en español se traduce como hoja de estilo en cascada, es un lenguaje de definición de estilos, en el cual se pueden definir variables y algunas funciones.

Sintáxis Básica

En CSS la sintaxis de las reglas constan de dos partes:

  1. El Selector: es el elemento que indica a que parte del documento HTML se le aplicará un bloque de declaración estilos. El selector puede ser:
    • Cualquier etiqueta HTML. Ejemplo h1, p, body, a, etc...
    • Una clase representado con el símbolo del punto (.) y una palabra cualquiera. Ejemplo .header, .bg-color, .ventana, etc...
    • Un id representado con el símbolo del numeral (#) y una palabra cualquiera. Ejemplo #header, #bg-color, #ventana, etc...

      • Tanto el selector de clase como el de id deben estar declarados como atributos dentro de la etiqueta al a que se quiere dar estilo

  2. El Bloque de Declaraciones: es cada uno de los atributos del elemento HTML que vamos a modificar. La sintaxis obligatoria es la siguiente: css Elemento-HTML { atributo: valor; atributo-de-mas-de-dos-palabras: valor; } /*Ejemplo*/ h1 { color: orange; background-color: antiquewhite; }

    El bloque de declaración va dentro de llaves, lo atributos de mas de una palabra van separados por guiones, luego de los atributos va dos puntos (:), seguidos del valor de dicho atributo y para finalizar hay que poner obligatoriamente punto y coma (;)

Comentarios en CSS

Los comentarios en CSS van de limitados por (/*) y (*/). Ademas de comentar también se usan para inhabilitar código.

css /*h1 { color: orange; background-color: antiquewhite; } */

Formas de aplicar CSS

Hay tres formas de invocar los estilos:

  • En línea: es cuando a través del atributo style de HTML aplicas los estilos directo al elemento. <h3 style="font-weight: bold;">Titulo de sección</h3>
  • Interno: es cuando aplicamos dentro el mismo documento HTML con la etiqueta style en la cabecera <head> <style> body { display: flex; flex-direction: column; } </style> </head>
  • Externo: es cuando en la cabecera del documento con la etiqueta <link> enlazamos a una hoja de estilos externa. <head> <link rel="stylesheet" href="styles.css"> </head>
  • @import url(): de esta manera podemos mandar a llamar una hoja de estilo desde otra hoja de estilo y aplicar los estilos de la hoja enlazada. Pero no es una buena practica porque este método es bloqueante.

Selectores CSS

Selectores básicos (etiquetas, id y class):

  • Etiquetas: son elementos del HTML. Este tipo de selector aplica el estilo a todos los elementos con esa etiqueta que estén dentro del documento HTML. css html { box-sizing: border-box; font-size: 16px; }
  • Identificadores: es el atributo id de un elemento HTML y se identifica con un numeral (#). Dar estilos con id se considera un AntiPatron, porque los id son únicos además tienen mayor peso que los otros selectores, por eso no es recomendado. css #navbar { display: none; position: fixed; top: 0; left: 0; min-width: auto; width: auto; height: 100vh; background-color: salmon; }
  • Clases: es el atributo class de un elemento HTML y se identifica con un punto (.). Las clases son las mas recomendadas porque pueden ser múltiples, se pueden repetir en distintas etiquetas HTML y son reutilizables. Es recomendable nombrar las clases según la función que cumplan. css .main-section{ padding: 1rem; }

Selector de hijos directos y descendientes

  • Un selector de hijos directos es aquel primer elemento anidado dentro de otro. Se representa con el símbolo (>). En el siguiente ejemplo seleccionaremos los hijos directos li del elemento ul que tiene la clase "hijos-directos". html <ul class="hijos-directos"> <li><b>Primavera</b></li> <li><b>Verano</b></li> <li><b>Otoño</b></li> <li><b>Invierno</b></li> </ul> css .hijos-directos > li { background-color: thistle; }
  • Un selector de hijos descendientes es aquel que se aplica a cualquier elemento sin importar el orden anidado dentro de otro. Se representa con un espacio en blanco ( ). html <ul class="hijos-directos"> <li><b>Primavera</b></li> <li><b>Verano</b></li> <li><b>Otoño</b></li> <li><b>Invierno</b></li> </ul> css .hijos-directos b { background-color: thistle; }

Selector de hermanos y adyacente

  • Los selectores hermanos en general aplica estilo a los elementos siguientes (que estén por debajo) en el mismo nivel del elemento de referencia. Se representa con el símbolo (~) html <ul > <li><b>Primavera</b></li> <li class="hermanos-general"><b>Verano</b></li> <li><b>Otoño</b></li> <li><b>Invierno</b></li> </ul> css .hermanos-general ~ li { background-color: thistle; }
  • Los selectores de hermanos adyacentes a diferencia del general, aplica solo al siguiente hermano. Se representa con el signo (+). html <ul > <li><b>Primavera</b></li> <li class="hermano-adyacente"><b>Verano</b></li> <li><b>Otoño</b></li> <li><b>Invierno</b></li> </ul> css .hermano-adyacente + li { background-color: thistle; }

Selectores de atributos

Los estilos CSS se pueden aplicar tomando como referencia cualquier atributo de cualquier etiqueta HTML.

  • Se puede acceder a los atributos con los corchetes junto al selector de la etiqueta []. css .selectores-atributos a[href] { color: orangered; }
  • También podemos acceder al valor del atributo declarando atributo y valor. css .selectores-atributos a[target="_blank"] { font-weight: bold; }
  • Otra manera de usarlo es pasando un (*) y un valor junto al atributo, esto buscara el valor pasado dentro del atributo. css .selectores-atributos a[href*="jonmircha"] { font-size: 24px; }
  • Similar a la forma anterior también podemos pasar un atributo unido al signo [^] y una valor, esto lo que hará es aplicar el estilo siempre y cuando el atributo empiece con el valor pasado. css .selectores-atributos a[href^="http:"] { text-decoration: none; }
  • Inverso al anterior, si usamos el signo ($) aplicará el estilo siempre y cuando el valor del atributo termine con el valor pasado. css .selectores-atributos a[href$=".com"] { background-color: brown; }
  • Usando el signo de elemento hermano (~) podemos buscar dentro de una lista de clases. css .selectores-atributos a[href$=".com"] { background-color: brown; }
  • El signo (|) similar al anterior, nos permite buscar dentro de una lista de clases, pero con la variante es que el valor buscado debe estar seguido por un guion medio. css .selectores-atributos a[class|="menu"] { font-style: italic; }

Selector Universal

El selector universal aplica estilos a todos los elementos HTML que hayan dentro del documento. Se representa con el símbolo (*).

html <ul class="selector-universal"> <li><b>Primavera</b></li> <li ><b>Verano</b></li> <li><b>Otoño</b></li> <li><b>Invierno</b></li> </ul> css * { font-family: "Franklin Gothic Medium", "Arial Narrow", Arial, sans-serif; } .selector-universal * { font-size: 26px; }

Agrupación de selectores

Para agrupar selectores solo debemos separarlos con coma (,) antes de los corchetes

css input[type="text"], input[type="email"], textarea { border-color: yellow; background-color: black; display: block; border-width: 2px; font-size: 20px; width: 300px; }

Pseudo-clases

Las pseudo-clases van a dar estilos dependiendo del contexto, posición o estado del elemento. Para usarlas se colocan dos puntos (:) después del selector. A continuación algunas de las mas usadas.

css pseudo-clases de estado .selector:required { border-color: yellow; } .selector:valid { border-color: green; } .selector:invalid { border-color: red; } .selector:focus { background-color: skyblue; } .selector:disabled { background-color: pink; } .menu a:visited { color: olive; } .menu a:hover { color: orange; } .p:hover { font-size: 20px; } #temario:target { background-color: olivedrab; } pseudo-clases de posición .lista li:first-child { ⬅️Primer elemento hijo color: green; } .lista li:last-child { ⬅️Ultimo elemento hijo color: violet; } .lista li:nth-child(3) { ⬅️Puede recibir la posicion del elemento hijo color: navy; } .lista li:nth-child(4n + 1) { ⬅️O un patron para aplicar estilos font-size: 30px; } .lista li:nth-child(3n) { ⬅️Un patron que estila cada 3 elementos hijos font-weight: bold; } .lista li:nth-child(even) { ⬅️Patron que aplica a los hijos pares background-color: orange; } .lista li:nth-child(odd) { ⬅️Patron que aplica a los hijos impares background-color: yellow; } pseudo-clases de contexto .articulo p:first-of-type { ⬅️Primer hijo de su tipo background-color: pink; } .articulo p:last-of-type { ⬅️Ultimo hijo de su tipo background-color: violet; } .articulo p:nth-of-type(3) { ⬅️Patron cada 3 elementos de su tipo background-color: fuchsia; } .articulo *:not(p, footer) { ⬅️Aplica a todos excepto a los que se le pasaron background-color: beige; }

Pseudo-elementos

Dan estilos a partes especificas de un elemento, se usa el (::) para diferenciarlos de las pseudoclases. Si queremos agregar contenido es importante usar el atributo content: “”;

css .saga-precio::before { ⬅️Agrega contenido antes del elemento content: "$"; font-weight: bold; } .saga-precio::after { ⬅️Agrega contenido después del elemento content: "USD"; } .input::first-letter { ⬅️Aplica a la primera letra del elemento font-weight: bold; font-size: 32px; } .input::first-line { ⬅️Aplica a la primera linea del elemento color: coral; } .input::selection { ⬅️Aplica cuando seleccionamos un texto color: green; background-color: blueviolet; } .input::placeholder { ⬅️Aplica al placeholder del input color: violet; }

El Algoritmo de CSS

El algoritmo de CSS es la forma en que el navegador aplica los estilos al documento HTML. Es vital entender este concepto par que entiendas como se aplican y en algunas ocasiones se sobrescriben las reglas CSS.

Cascada

Es el mecanismo que tiene el navegador web para ir aplicando los estilos, y en base a esto toma tres aspectos en particular:

  1. Origen del Código

    • Primero tenemos el userAgent, son los estilos que se aplican por defecto al documento.
    • Después tenemos las personalizaciones del usuario, por ejemplo el zoom que el usuario hace a una pagina
    • Por ultimo el código del autor, que básicamente son los estilos que le escribamos al documento.

  2. Especificidad del Selector

    La especificidad es el peso que tiene un selector cuando hay conflicto de estilos. Se calcula de la siguiente forma:

    • Etiquetas y Pseudoelementos ➡️ 0,0,0,1
    • Clases, Atributos y Pseudoclases ➡️ 0,0,1,0
    • Identificadores ➡️ 0,1,0,0
    • Estilos en línea ➡️ 1,0,0,0
    • !important ➡️ Rompe la especificidad priva sobre todos

  3. Herencia

    La herencia es la capacidad de un selector de obtener (heredar) los valores de sus ancestros mas cercanos, para aplicarla se usa el valor inherit, si queremos evitarla podemos asignar otro valor o inicializar la propiedad en cuestión con el valor initial. No todos los valores son heredables automáticamente, en general los que mas se heredan automáticamente son los valores que conciernen a la tipografía.

Reseteo y Normalización de Estilos

Podemos usar hojas externas como normalize.css que nos brindan un “reseteo” del css para luego trabajar sobre el.

Prefijo de los navegadores

En el tiempo en el que los navegadores no se actualizaban tan a la par existían unos prefijos que nos ayudaban a implementar elementos del lenguaje que aun no estaban plenamente soportadas por todos los navegadores. Pero estos prefijos ya no son tan necesarios. Los prefijos son los siguientes:

  • Navegadores de Microsoft: -ms-
  • Navegadores de Mozilla: -moz-
  • Navegadores de Opera: -o-
  • Todos los demás navegadores: -webkit-

Modelo de Caja

Es la forma en como CSS interpreta las etiquetas HTML, pues para CSS cada una de las etiquetas HTML son cajas, por ser una cajas tiene 4 lados que van en el siguiente orden: top, right, bottom y left. Las cajas pueden ser en línea o en bloque con las siguientes propiedades:

Content

Es el contenido del elemento HTML, tomando en cuenta sus dimensiones (width & height). Estas dimensiones hacen referencia al ancho y alto del modelo de caja y se ven influida por el tamaño del viewport, pero pueden ser modificadas.

Border

Es lo que delimita cada uno de los elementos HTML. Las propiedades de border son:

  • border-color: manipula el color del borde.
  • border-width: manipula la anchura del borde, se le puede pasar unidades de medida o palabras como thin → 1px; medium → 3px o thick → 5px.
  • border-style: manipula el estilo del borde, los mas populares son solid, dashed o dotted.
  • border-radius: es un shorthand que manipula cuan redondeada pueden ser las esquinas de una caja, el orden en que va aplicando el estilo es arriba-izq, arriba-der, abajo-der y por ultimo abajo-izq. Si queremos manipular una por una podemos usar border-top-left-radius, border-top-right-radius, border-bottom-left-radius, border-bottom-right-radius. Tambien podemos elegir poner border elípticos en vez de redondeados agregando una barra border-radius: 2rem / 4rem;. Si queremos un circulo, podemos pasarle un valor de 50% a border-radius
  • outline: es un propiedad que agrega un borde adicional pero no entra en el modelo de caja, es decir no interfiere con el espacio de los otros elementos. Recibe al igual que border, grosor, estilo y color. También tiene una propiedad llamada outline-offset que aleja el outline del border, hacia afuera o hacia adentro. Es comunmente usado para aplicar efectos de sombreado.

Padding y Margin

El padding son las distancias internas que hay desde el borde hasta el contenido, y el margin son las distancias externas que hay entre el elemento y sus hermanos o padres. Para ambas propiedades los valores se pueden pasar de la siguiente manera:

  • 1 valor: aplica para los 4 lados por igual.
  • 2 valores: el primer valor aplica para los lados verticales top y botton, y el segundo valor aplica para los lados horizontales right y left.
  • 3 valores: el primer valor aplica para top, el segundo para los dos lados horizontales y el tercero para el bottom.
  • 4 valores: cada valor aplica para cada lado de la caja, en el orden antes mencionado.

Un dato importante al momento de maquetar es que los márgenes verticales se colapsan o se superponen, por tal razón si es recomendable usar o margin-top o margin-bottom. Esto solo aplica para los margenes verticales, los margenes horizontales si se suman.

Cajas de Línea vs Cajas de Bloque

Caja de Línea

  • Ocupan el espacio necesario para mostrar su contenido.
  • No tienen dimensiones modificables (alto, ancho).
  • Permiten otros elementos a su lado.
  • Padding y margin solo empujan a elementos adyacentes en horizontal, nunca en vertical.
  • Los elementos en línea son básicamente el span y los que afectan el texto, también los multimedia, y los elementos del formulario.

Caja de Bloque

  • Ocupan todo el ancho disponible, lo que genera saltos de línea.
  • Tienen dimensiones modificables (alto y ancho).
  • No permiten otros elementos a su lado, (aunque especifique un ancho, siguen ocupando todo el espacio disponible).
  • Los elementos en bloque son básicamente el div, todos los encabezados, el formulario, entre otros.

Propiedad Display

Es la propiedad que permite visualizar una caja. Tiene las siguientes valores:

  • inline: propiedad por defecto de las cajas de linea
  • block: propiedad por defecto de las cajas de bloque
  • inline-block: (se comporta como una caja de línea per acepta modificar sus dimensiones de alto y ancho)
  • none: oculta la caja, pero sigue existiendo en el documento
  • list-item: propiedad por defectos de las listas, aplica la sangría características de estas.
  • table: propiedad por defectos de las tablas.
  • flex
  • grid

Existe una propiedad similar a display: none, que se llama visibility cuya diferencia radica en que con visibility, podemos ocultar el elemento pero conservando el espacio de maquetado. Los valores de visibility son hidden y visible

Propiedad Overflow

Esta propiedad nos permite definir que sucederá con el contenido cuando este desborde su caja contenedora. Tiene las siguientes propiedades:

  • visible: el contenido puede desbordar la caja contenedora.
  • hidden: oculta todo el contenido que no quepa en la caja contenedora.
  • scroll: aplica barras de desplazamiento para poder leer todo el contenido.
  • auto: esta propiedad aplica barras de desplazamiento según necesite el contenido.

Tambien existen overflow-x y overflow-y para aplicar la propiedad overflow a cada eje de la caja por separado.

Tamaño de Caja

El box-sizing define como se calcula el ancho y la altura de una caja. Tiene dos propiedades:

  • content-box: esta es la propiedad por defecto, y lo que indica es que el tamaño de la caja empieza a partir del content box, sin incluir margenes y padding. Es decir si pasamos un width de 200px, vamos a tener como resultado un content box de 200px y a esto hay que sumarle lo que mida el border y el padding.
  • border-box: indica es que el tamaño de la caja empieza a partir del border de la caja, incluyendo margenes y padding.

Es buena practica resetear el documento para que todo sea border-box.

/* apply a natural box layout model to all elements, but allowing components to change */ html { box-sizing: border-box; } *, *:before, *:after { box-sizing: inherit; }

Centrado de Caja

Para el centrado de cajas es recomendable usar las propiedades margin-left: auto y margin-right: auto;

Posicionamiento en CSS

La propiedad position al igual que la propiedad float nos permiten modificar la posición natural de cualquier elemento del documento HTML.

El navegador coloca cada elemento teniendo en cuenta el orden en el que aparece en el documento y su tipo de visualización dependiendo si es un elemento de línea o de bloque. Con estas propiedades podemos alterar este comportamiento.

Position trabaja con las propiedades top y left, bottom y right, y z-index (profundidad). Las propiedades top y left van a tener preferencia sobre bottom y right, es decir o usamos el primer par o el segundo. Estos parámetros aceptan valores negativos. Cuando usemos la propiedad static estos ejes no van a funcionar.

Posicionamiento static

Es la propiedad por defectos de los elementos HTML, cuando está especificada, las propiedades de top, left, right o bottom no funcionan.

Posicionamiento relative

Con el posicionamiento relative el elemento conserva sus dimensiones y posición original en el flujo del documento. Si se mueve toma como referencia su posición original. Es decir, a partir del espacio que ocupe originalmente se va a mover según la indicación dada.

Posicionamiento absolute

Con el posicionamiento absolute el elemento pierde sus dimensiones y posición original en el flujo del documento. Si se mueve puede tomar como referencia 2 elementos:

  • El primer ancestro con posicionamiento relative.
  • Si no encuentra al ancestro, se mueve respecto al documento HTML.

Posicionamiento fixed

Con fixed el elemento pierde sus dimensiones y posición original en el flujo del documento. Si se mueve toma como referencia el documento HTML y queda fijo en la posición, cuando el scroll se mueve, elemento no lo hará, quedará fijo.

Posicionamiento sticky

El posicionamiento sticky es una combinación de relative y fixed, para este posicionamiento las propiedades de top, left, bottom y right no mueven el elemento, sino que sirven como un punto de referencia, mientras no lleguen a ese valor el elemento se comporta como relative, cuando se llega, se convierte en fixed.

Para que este comportamiento funcione el elemento sticky debe ser hijo directo del body o su elemento contenedor debe tener dimensiones definidas y solo será sticky dentro de las dimensiones del su contenedor padre.

Propiedad z-index

Es la propiedad que permite controlar la profundidad de los elementos posicionados. Su valor por defecto es auto, acepta números positivos y negativos. A mayor valor del elemento está mas al frente, a menor valor mas al fondo.

Un elemento padre nunca podrá estar sobre sus elementos hijos, sin embargo los elementos hijos si pueden, dándoles un valor negativo y que el elemento padre no tenga definido un z-index. Es aconsejable ir definiéndonos de en unidades de 100 en 100.

Los elementos flotantes

  • float: convierte un elemento en flotante, permitiendo que sea empujado al espacio mas cercano que encuentre a la derecha o la izquierda respecto a la posición que ocupaba en el flujo de los elementos del documento HTML. Esto automáticamente lo saca del flujo del cuerpo del documento, pudiendo generar problemas de maquetado.
  • clear: viene a arreglar o limpiar los problemas en el flujo de los elementos del documento HTML ocasionados por float.(left, right, both)
